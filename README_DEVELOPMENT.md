# План поэтапного развития сайта

Документ для **понемногу**: небольшие шаги, каждый можно закрыть отдельным коммитом. Ориентиры — рекомендации из папки **«требования на сайт»** и стек **Flask + MySQL + Jinja2**.

## Уже есть в коде (не дублируем в чек-листе)

- Шапка с текстовым названием (`h1`), логотип, поиск → `/search`, **RU / BY / EN** (cookie и `?lang=`), новости из БД на **трёх языках** (поля `news.*`).
- Кнопка доступности, модалка, **localStorage**; футер с контактами, соцсетями, копирайтом, баннерами партнёров (**`rel="noopener noreferrer"`** на внешних ссылках).
- Единый **`lang="{{ current_lang }}"`** в `base.html`; **hreflang** (`ru` / `be` / `en`, `x-default`).
- Карточки специальностей `/spec/*`, статика через **`/static/`** (в т.ч. **`static/js/`**).
- **JSON API** ` /api/v1` для отдельной админки (ключ **`X-API-Key`**, см. `README.md`).
- **`pytest`** в каталоге `tests/`.
- **Страница 404** — только блок ошибки; партнёры, футер и модалка доступности подключаются один раз через `footer.html` (без дубля). В `footer.html` **нет** лишних `</body></html>` — документ закрывается в `base.html`.
- **Этап 1 (технически):** в футере блок **«Документы и сервисы»** — ссылки на **`/privacy`**, **`/one-window`**, **`/sitemap`**, **`mailto:`** администратору (`site.admin_email` в `config.yml`). Строка **ГРИС** из `site.gris` или заглушка (`show_placeholder_if_empty`). Обработчики в `page/site_pages.py`.

---

## Как пользоваться

1. Идите **по этапам сверху вниз** внутри фазы.
2. После шага — проверка в браузере и короткий коммит.
3. Контентные шаги согласуйте с администрацией колледжа.

## Дальше — вне жёсткого чек-листа (имеет смысл после интеграции админки)

Эти пункты дополняют план, когда основной контент и юридические блоки закрыты:

| Направление | Зачем |
|-------------|--------|
| **Связка админки с API** | Сценарии создания/редактирования новости, обработка ошибок и синхронизация кэша (если появится). |
| **CORS** | Только если админка в браузере на другом origin; иначе вызовы API с сервера админки. |
| **OpenAPI (Swagger)** | Контракт для команды админки и автогенерация клиента. |
| **`<link rel="canonical">`** | Рядом с hreflang — борьба с дублями URL с разными `?lang=` и параметрами поиска. |
| **Загрузка файлов** | Отдельный эндпоинт или S3/хостинг картинок вместо ручного `image_path`. |
| **Лимиты и аудит API** | Rate limit по IP/ключу, логирование для расследований. |
| **`updated_at` у новостей** | Для инкрементальной синхронизации с админ-сервисом. |
| **Вынос блока партнёров** | При необходимости один раз подключать `{% include "partners_block.html" %}` из футера (сейчас блок один). |
| **E2E-тесты** | Playwright/Selenium на критические сценарии (язык, поиск, 404). |
