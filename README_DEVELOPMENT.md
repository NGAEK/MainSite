# План поэтапного развития сайта

Документ для **понемногу**: небольшие шаги, каждый можно закрыть отдельным коммитом. Ориентиры — рекомендации из папки **«требования на сайт»** и стек **Flask + MySQL + Jinja2**.

**Уже есть в коде (не дублируем в чек-листе):** копирайт и блок контактов в футере, режим работы, соцсети, баннеры партнёров (в т.ч. president.gov.by, edu.gov.by и др.), шапка с **текстовым** названием (`h1`) и логотипом, поиск в хедере → `/search`, переключение **RU/BY** (cookie / `?lang=`), кнопка доступности (контраст, крупный шрифт, тёмная тема) с **сохранением в localStorage**, адаптивное меню с выпадающими подпунктами, ссылка **«Главная»** на `/`, лента **новостей** на главной из БД, карточки **специальностей** `/spec/*`, раздача `/static` и `/js`.

---

## Как пользоваться

1. Идите **по этапам сверху вниз** внутри фазы.
2. После шага — проверка в браузере и короткий коммит.
3. Контентные шаги согласуйте с администрацией колледжа.

## Этап 4. Контентные разделы (по одному)

1. **Об учреждении** — реальные страницы вместо `#`: история, администрация, документы и т.д.
2. **Абитуриентам** — сводная страница **«Специальности»** с таблицей/списком и ссылками на существующие `/spec/*`.
3. **Контакты** — отдельная страница: полный адрес с индексом, телефоны в формате **+375 …**, кликабельные **`tel:`** и **`mailto:`**, при необходимости карта проезда; согласовать с блоком в футере (один источник правды).
4. Остальные рубрики из методрекомендаций (учащимся, методка, идеология, ЦК…) — по согласованию, отдельными задачами.

**Технически:** новые `page/*.py`, маршруты в `app.py`, шаблоны; при необходимости — правки `ngaek.sql` / миграции.
